Требуется создать CLI-инструмент, который запускает кэширующий прокси-сервер. Он будет перенаправлять запросы к реальному серверу и кэшировать ответы. Если тот же запрос поступает снова, сервер вернёт кэшированный ответ вместо перенаправления запроса на сервер.

# Требования

Пользователь должен иметь возможность запустить кэширующий прокси-сервер командой следующего вида:

```shell
caching-proxy --port <number> --origin <url>
```

`--port` — порт, на котором будет работать кэширующий прокси-сервер.

`--origin` — URL сервера, которому будут перенаправляться запросы.

Например, если пользователь выполнит следующую команду:

```shell
caching-proxy --port 3000 --origin http://dummyjson.com
```

Кэширующий прокси-сервер должен запуститься на порту 3000 и перенаправлять запросы на `http://dummyjson.com`.

В приведённом выше примере, если пользователь сделает запрос на `http://localhost:3000/products`, кэширующий прокси-сервер должен перенаправить запрос на `http://dummyjson.com/products`, вернуть ответ вместе с заголовками и кэшировать ответ. Также добавьте заголовки в ответ, которые указывают, получен ли ответ из кэша или с сервера.

```
# Если ответ из кэша
X-Cache: HIT

# Если ответ с исходного сервера
X-Cache: MISS
```

Если тот же запрос поступает снова, кэширующий прокси-сервер должен вернуть кэшированный ответ вместо перенаправления запроса на сервер.

Также необходимо предусмотреть способ очистки кэша командой следующего вида:

```shell
caching-proxy --clear-cache
```

После создания описанного выше проекта вы получите хорошее понимание того, как работает кэширование и как можно создать кэширующий прокси-сервер для кэширования ответов от других серверов.
